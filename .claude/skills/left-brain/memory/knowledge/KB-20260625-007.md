# autonomous-runner 修复 #1 后发现的 #2 BUG

## 现象
2026-06-25 16:22 修复 #1（spawn ENOENT）后开 single 模式跑 runner：
- spawn 真起来了：bin = `C:\Users\Administrator\AppData\Roaming\npm\claude.cmd` ✅
- 子会话结束 code=0 ✅（真跑了）
- 但 runner 仍判定"失败 (6/5)"：子进程退出 code=-1 ❌
- 根因：stage.status 仍是 `in_progress`，子会话没调 `complete-stage`

## 子会话的输出（节选）
```
你的消息"你正在执行"看起来是**半句话**——是要我做什么？...
1. 帮你整理/拆分这 34 个未提交改动？
2. 继续某个未完成的工作（M12 之后的下一个增量）？
...
告诉我具体目标就开干。
```
子会话把"prompt 里的 '你正在执行 AiCode 自主演进模式的一个阶段'"误读成了"用户的半句话指令"，然后走 SessionStart 启动协议 + 等用户补充。

## 根因
buildStagePrompt 设计不严：
- 第 2 步写"完成阶段工作"——模糊
- 第 3 步写"完成后必须执行 complete-stage"——但"完成后"被理解为可选
- 缺少 ⚠️ 强制标记（如"第一步必须是 ..."）
- 缺少自检提示（"调完 complete-stage 后再退出，否则 stage 会被判失败"）

子会话的行为逻辑更倾向"读上下文 + 询问用户"，而不是"按 prompt 清单执行"——这是 `claude -p` 的设计倾向。

## 修复方向（待定）
**选项 A（推荐）：prompt 开头加强制指令**
```
⚠️ 强制任务清单（按顺序执行，不可跳过）：
1. **第一件事**：执行 `node autonomous-runner.js complete-stage "<下一阶段>"` 标记阶段开始
2. 读 latest_state.json 拿完整上下文
3. 完成 ${stageName} 的开发工作
4. commit 改动
5. 再次执行 `complete-stage` 标记完成
6. bash session-summary.sh save 存快照
7. 直接退出
```

**选项 B：改造为 hook-driven**
- runner 不依赖子会话自觉调 complete-stage
- 通过 snapshot 时间戳 + 阶段状态推断（如：子会话退出后写个 marker，runner 检测到就视为成功）

**选项 C：妥协方案**
- runner 检测到 code=0 但 stage.status 未完成 → 自动 mark completed（不严格但实用）
