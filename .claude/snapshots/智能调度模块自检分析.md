### 详细描述

#### 1. [P0] 灰区任务被完全丢弃，无任何反馈机制
- **问题**: 第 188-195 行返回 `dispatch: null`（灰区）后，调用方需自行判断
- **影响**: 用户输入"修一下登录"这种模糊任务时，行为完全依赖调用方策略
- **建议**: 增加 `suggested_action` 字段：灰区时返回 `{action: "dispatch", agents: 2, hint: "建议派 2 个 agent 保守处理"}`

#### 2. [P0] `estimateFileCount` 关键词权重互相覆盖
- **问题**: 第 56-66 行用 `Math.max(estimate, weight)`，导致"全栈"覆盖"前后端"
- **影响**: 类似"前端 React + 后端 Spring"的任务会被算成 2 个文件，无法触发派代理
- **建议**: 改为累加或显式优先级表：`full_stack: 5, frontend_backend: 4, multi_module: 3`

#### 3. [P1] `detectTaskType` 缺少英文模式
- **问题**: 第 88-99 行正则全中文
- **影响**: "Please refactor this file" 误判率高
- **建议**: 加 `if (/refactor/i.test(text)) return 'refactor'`；加 `deploy|rollback|migrate` 模式

#### 4. [P1] `dont_dispatch.keywords` 缺口语化中文动词
- **问题**: 第 14-18 行只有"快速/简单/小/只改"
- **影响**: "帮我看下这段代码"会被当成派代理任务
- **建议**: 扩充：`['看下', '瞄一下', '瞄瞄', '扫一眼', '聊聊', '说说', '讲讲']`

#### 5. [P1] 规则无版本号/变更日志
- **问题**: 第 10-36 行的 `RULES` 硬编码
- **影响**: 无法回溯哪条规则在哪个版本生效
- **建议**: 加 `RULES.version = '1.0.0'`，文件头加 `@since / @changed` 注释

#### 6-8. P2 性能/体验类（详见 v1.2-改进清单.md）
