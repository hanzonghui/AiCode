## 📖 文件读取策略

- **按需读取** — 用户提到哪个文件就读哪个，不提前遍历整个目录
- **批量读取前确认** — 需要读多个文件时，先列清单让用户确认再逐个读取
- **优先搜索定位** — 使用 Grep/Glob 搜索而非盲目遍历目录
- **长篇只读关键部分** — 大文件只读取需要的行号范围
- **全局扫描前必须询问** — 严禁用 `find`、`glob **` 等方式扫描整个仓库；确需全局扫描时，先向用户说明原因并请求授权

## ✍️ 输出控制

- **回答精炼** — 避免冗长的解释性输出
- **代码直接可用** — 不输出大段注释的"教学式"代码
- **默认最简方案** — 用户没特别要求时，给最简洁的方案

## 🔄 任务切换

- 全新任务时建议 `/clear` 清空旧上下文
- **new task = new session**，除非强关联

## 🤖 子代理自主调度

当任务符合以下**任意一条**时，自动拆分子代理并行处理，用户无需手动指定：

| 触发条件 | 说明 |
|:---------|:------|
| 涉及 5+ 个文件的新建/修改 | 拆出编码子代理，主 context 只做协调 |
| 前后端 + 数据库全栈 | planner 规划 → 后端/前端子代理并行编码 |
| 包含"调研/研究/比较"等探索性任务 | explorer 单独调研，主 context 继续工作 |
| 用户说"复杂"、"全面"、"完整"等词 | 默认启用子代理模式 |

### 调度原则

- **planner** → 架构设计、技术选型、方案对比
- **explorer** → 代码搜索、文档调研、目录结构理解
- **reviewer** → 代码审查、Bug 发现、安全检查
- **并行**：不互相依赖的任务同时跑，互不阻塞
- **结论返回**：子代理只返回结果，中间过程不污染主 context
