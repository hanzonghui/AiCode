name: code-review
description: 多 Agent 代码审查 — 审查变更、发现 Bug、检查质量
---

执行完整代码审查流程：

1. **获取变更** — 读取当前 git diff（未提交的变更）：
   ```
   git diff
   git diff --cached  (如果有暂存的变更)
   ```

2. **审查维度**：
   - 逻辑 Bug — 空指针、竞态条件、错误处理、边界条件
   - 安全隐患 — SQL注入、XSS、敏感信息泄露
   - 可维护性 — 命名、注释、设计模式、重复代码
   - 测试覆盖 — 关键路径是否被测试覆盖

3. **输出格式**：
   ```
   📋 审查报告
   ━━━━━━━━━━━━━━━━━━━━━━━━━━━

   🔴 严重 (x项)
   - 描述 | 文件:行 | 修复建议

   🟡 警告 (x项)
   - 描述 | 文件:行 | 修复建议

   💡 建议 (x项)
   - 描述 | 文件:行 | 修复建议
   ```

4. **如果有严重问题**，询问是否修复后再继续
