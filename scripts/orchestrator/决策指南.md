# 智能调度决策指南（主会话版）

> 主会话（我）在每次回复用户前，**手动调用 dispatcher.js** 判断是否需要派子代理。
> 决策依据：本指南 + dispatcher.js 规则引擎

---

## 🚦 决策流程

```
用户提问
   ↓
我（主会话）调用 dispatcher.js
   ↓
获得决策结果（dispatch / agents / reason）
   ↓
┌─────────────────────────────────────────────┐
│ dispatch=true                                │
│   → 用 Agent 工具派 N 个子代理并行          │
│   → 等待全部完工                             │
│   → 汇总结果回复用户                         │
├─────────────────────────────────────────────┤
│ dispatch=false                               │
│   → 主会话直接答，不派                       │
├─────────────────────────────────────────────┤
│ dispatch=null（灰区）                        │
│   → 主会话自己判断                            │
│   → 默认按"值得派"处理（保守策略）           │
└─────────────────────────────────────────────┘
```

---

## 🎯 调用方式

### 方式 1：CLI（我每次回复前手动跑）

```bash
node H:/AI-han/AiCode/scripts/orchestrator/dispatcher.js "<用户原话>"
```

输出 JSON：
```json
{
  "dispatch": true,
  "agents": 2,
  "reason": "预估涉及 2 个模块",
  "layer": 1,
  "confidence": 0.9
}
```

### 方式 2：作为 Node 模块引用

```javascript

const decision = decide(userPrompt);
```

## 🤖 派 Agent 的执行模板

**当决策是 dispatch=true 时**，我按下面模板执行：

### 派 2 个 Agent（最常见）

```python
# Agent 1: 后端排查
Agent(subagent_type="Explore", prompt="""
排查 ${TASK} 的后端代码：
- 重点查看 Controller / Service / DAO 层
- 列出相关文件路径
- 输出可疑代码片段
""")

# Agent 2: 前端/数据排查
Agent(subagent_type="Explore", prompt="""
排查 ${TASK} 的前端代码：
- 重点查看 Vue/React 组件
- 同时检查数据库表结构、Redis 缓存
- 输出可疑代码片段
""")

# 主会话等两个 Agent 完工后汇总
```

### 派 3 个 Agent（大型任务）

```python
# Agent 1: 代码层
Agent(subagent_type="Explore", prompt="分析代码...")

# Agent 2: 数据层
Agent(subagent_type="Explore", prompt="分析数据库/缓存...")

# Agent 3: 配置/环境层
Agent(subagent_type="Explore", prompt="检查 .env / 配置文件...")
```

## 📊 决策结果对照表

| dispatch | 含义 | 我的行动 |
|:---------|:-----|:---------|
| `true` + agents=2 | 派 2 个子代理并行 | 用 Agent 工具派发，等完工汇总 |
| `true` + agents=3 | 派 3 个子代理并行 | 用 Agent 工具派发，等完工汇总 |
| `false` | 不派 | 主会话直接答 |
| `null` | 灰区 | 主会话自行判断，默认倾向派 |

## ⚠️ 注意事项

1. **用户感知为零**：钩子不暴露决策给用户，决策只在主会话内部执行
2. **子代理使用 Explore 类型**：只读，不修改文件，安全
3. **结果汇总由主会话完成**：子代理输出 + 主会话综合分析
4. **成本控制**：当前不设上限，监控 token 用量后再调整
5. **规则可热更新**：改 `dispatcher.js` 后立即生效，无需重启会话
