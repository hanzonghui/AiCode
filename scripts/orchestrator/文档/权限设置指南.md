## ⚠️ 安全说明

### 这些规则允许什么？

| 规则 | 允许的操作 |
|:-----|:-----------|
| `Write(.claude/commands/**)` | 写入 `.claude/commands/` 下的所有文件（创建/修改斜杠命令） |
| `Write(.claude/hooks/**)` | 写入 `.claude/hooks/` 下的所有文件（创建/修改钩子） |
| `Edit(.claude/settings.local.json)` | 编辑 `.claude/settings.local.json` 文件 |
| `Edit(.claude/agents/**)` | 编辑 `.claude/agents/` 下的所有文件 |
| `Bash(mkdir -p .claude/**)` | 创建 `.claude/` 下任意子目录 |
| `Bash(cp scripts/orchestrator/commands/*.md .claude/commands/)` | 复制 orchestrator 命令到 .claude/commands/ |
| `Bash(node H:/AI-han/AiCode/scripts/orchestrator/dispatcher.js *)` | 跑 dispatcher 规则引擎 |
| `Bash(node H:/AI-han/AiCode/scripts/orchestrator/test-dispatcher.js)` | 跑 dispatcher 测试 |

### 风险点

- `Edit(.claude/settings.local.json)` 让 AI 可以改权限配置 → **但** 改完还要过你这一关（你随时可以撤销）
- 写 `commands/**` 和 `hooks/**` → **AI 可以定义新行为**，但都在你眼皮底下（CC 会展示命令内容）

### 进一步加固（可选）

如果想更严，可以把 `Edit(.claude/settings.local.json)` 拆成更细的规则，比如：

```json
"Edit(.claude/settings.local.json permissions.allow)",  // 只能加 allow 规则
```

但这个语法 CC 可能不支持，需要测试。

## ❓ FAQ

### Q1：为什么 auto mode 不让我自己加？

**A**：防止 AI 给自己"提权"。这是**正确的安全设计**。权限必须由人类授权。

### Q2：加完会不会有安全风险？

**A**：写 `.claude/commands/` 的命令 CC 会在调用前**展示内容**，你随时可以拒绝。改 `settings.local.json` 也会保留你的**最终决策权**。

### Q3：能不能用更宽松的通配符？

**A**：不建议。比如 `Bash(node **/*.js *)` 等于"跑任何 .js"，**太宽**。精确到具体脚本更安全。

### Q4：能不能"全部允许"？

**A**：技术上可以加 `"Bash(*)"`，但**这等于禁用所有安全检查**。建议保持最小授权。

### Q5：项目级 vs 用户级怎么选？

**A**：
- 你**只在这个项目**用 → 项目级
- 你**所有项目**都要这个能力 → 用户级
- **建议先用项目级**验证效果
