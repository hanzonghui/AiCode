## 💡 决策依据

**为什么 v1.2 批次 1 这 6 项？**

1. **互相印证**：Agent 1（dispatcher 自身） + Agent 2（4 工具）都提到的 → 高置信度
2. **工作量小**：单项 < 30 min，2.5h 内能完成
3. **风险低**：不破坏 12/12 测试
4. **价值高**：直接影响日常使用体验

**为什么排除 LLM 接入？**

- Claude Code 单会话单模型，要打分需要外部 LLM（OpenAI/Ollama）
- 增加网络依赖和延迟（钩子 3s 超时）
- 启发式已经够用（12/12 100% 准确率）
- 等真实需求出现再上

**为什么排除趋势图/实时监控？**

- 当前只有 5 条决策日志
- 数据量不够画图
- 等积累 50+ 条再说
