# AI 自我约束规范（智能调度模块）

> **作用**：让 AI 在完成改动后**自动**保存快照、更新文档、写 KB，**不需要用户提醒**。
> **位置**：`scripts/orchestrator/自我约束规范.md`
> **最后更新**：2026-06-22

---

## 🎯 核心原则

**完成改动 = 触发自动收尾流程**，不需要用户说"保存一下"或"更新文档"。

---

## 📊 改动分级（决定触发什么）

| 级别 | 例子 | 触发动作 |
|:-----|:-----|:---------|
| **🟢 微小** | 修 typo、注释、空行 | 1（测试通过即可） |
| **🟡 小** | 修 bug、加参数、加日志 | **0 先存快照** + 1 + 2 |
| **🔴 大** | 新功能、新模块、架构变化 | **0 先存快照** + 1 + 2 + 3 + **4a 同步根目录 4 文档 + CHANGELOG（见 `.claude/rules/doc-sync.md`）** |
| **🏁 里程碑** | v1.2 全部完成、版本发布 | **0 先存快照** + 1 + 2 + 3 + **4a** + 5 |

**动作编号**：
0. **🆕 commit 前自动存快照**（[2026-06-25 强化](.claude/rules/self-discipline.md)）：任何 🟡 / 🔴 / 🏁 级别改动，**commit 前**必须先跑 `bash .claude/skills/left-brain/scripts/session-summary.sh save "..." -m "下一步"`。原因：milestone 模式按"完成/里程碑"标签触发，docs/refactor 类型 commit 不带这些标签会全跳过。
1. 跑测试（test-dispatcher.js + test-e2e.js）
2. 存快照（save.js，受 `.claude/snapshot-config.json` 控制频率）
3. 写 KB（left-brain.sh remember）
4. 更新文档（精简版+详细版的对应章节）
4a. **🔴 大 / 🏁 强制同步** 5 个根目录文档：见 `.claude/rules/doc-sync.md`
   - `04_自我演进路线.md`（0.4 节增量状态 / 0.5 节 L 表 / 第十二章里程碑表 / 顶部日期）
   - `03_版本迭代计划.md`（第四节 P0/P1 状态 / 第七节发版流程）
   - `02_工作空间功能介绍.md`（仅当用户可见变化时）
   - `CLAUDE.md`（工作空间结构 / 能力表 / 快速操作）
   - `CHANGELOG.md`（**每次必写** Unreleased 段）
5. 全局归档（global-archive.sh，可选）

> **快照配置说明**：默认 `mode: milestone`，只有完成/里程碑类标签才会自动保存，普通小改动不会频繁生成快照。可手动调整 `.claude/snapshot-config.json`。

## 🔄 完整自动收尾流程

```
完成改动（用户说"做完了"/"完成"/"OK"）
       ↓
【第 1 步：判断级别】用 EnterPlanMode 评估（如果级别不明确）
       ↓
【第 2 步：跑测试】
node scripts/orchestrator/test-dispatcher.js
node scripts/orchestrator/test-e2e.js
       ↓
【第 3 步：判断测试结果】
   通过 → 继续
   失败 → 不要存快照，先修 bug
       ↓
【第 4 步：存快照】
node scripts/会话快照/save.js "<改动标题>" "<标签>" -m "<下一步>"
       ↓ save.js 自动追加 00_ROOT_快速加载会话.md
       ↓ 自动检测"完成/里程碑/交付"标签 → 加三级检查点提示
       ↓
【第 5 步：判断是否需要更新文档】
   改动 ≥ 1 个章节 → 更新精简版+详细版
   改动 < 1 个章节 → 跳过
       ↓
【第 6 步：写 KB】
bash .claude/skills/left-brain/scripts/left-brain.sh remember "<事实摘要>"
       ↓
【第 7 步：判断是否需要全局归档】
   是里程碑（v1.X 全部完成）→ bash scripts/parallel/global-archive.sh "<任务名>"
   不是 → 跳过
```

## 🎯 详细规则

### 何时更新文档

**✅ 要更新**：
- 加了新功能（如 v1.2 加了灰区 suggested_action）
- 加了新模块（如 v1.3 加了三级检查点）
- 改了 API/接口（如 save.js 加了 -m 参数）
- 改了 RULES.version
- 改了核心架构（如 dispatcher.js 改成 YAML）

> **🔴 大 / 🏁 级别强制动作 4a**（不可跳过）：
> 同步 `.claude/rules/doc-sync.md` 中规定的 4 文档 + CHANGELOG。
> 这是为了避免"已完成但文档还写计划中"的状态漂移——
> 04 文档的 L4/L5 段曾出现"✅ 已达"但其实只跑了 M7 的情况，6/25 用户已经指出该问题。
> 具体同步清单和操作格式 → `.claude/rules/doc-sync.md`。

**❌ 不更新**：
- 修了个 typo
- 加了 1 行注释
- 改了 log 输出格式
- 内部重构（用户感知不到）

### 何时写 KB

**✅ 要写**：
- 改了默认行为
- 加了新约定/约束
- 做了新决策
- 发现新 bug
- 改了流程

**❌ 不写**：
- 微小 typo
- 临时调试

### 何时存快照

**✅ 要存**（任何 🟡 / 🔴 / 🏁 改动，**commit 前必跑**）：

```bash
# commit 前必跑（🔴 大 / 🏁 里程碑级别强制，🟡 小级别建议）
bash .claude/skills/left-brain/scripts/session-summary.sh save "[已完成] <一句话摘要>；下一步: <下一选题>" -m "next: <下一阶段>"
```

**频率**：
- 🟡 小：1 commit 1 快照
- 🔴 大：1 commit 1 快照（强制）
- 🏁 里程碑：1 里程碑 1 全局归档（global-archive.sh）+ 1 commit 1 快照

**❌ 不存**：
- 调试中的中间状态
- 测试都没跑过的改动
- 已经在 30 分钟内存过且未发生新 commit

### 何时归档

**✅ 要归档**：
- 里程碑完成（v1.X 全部完成）
- 一个独立任务全部结束
- 想做"完成报告"

**❌ 不归档**：
- 日常小改动
- 中间过程

## 🚨 反例（不要做）

- ❌ 用户没要求**就不要**改文档
- ❌ 不要因为"改了一行代码"就大改精简版
- ❌ 不要在测试没跑过之前存快照
- ❌ 不要存空白或无意义的快照

## 🧠 决策树

```
改动完成？
  ├─ 跑测试
  │   ├─ 失败 → 修 bug
  │   └─ 通过 ↓
  │
  ├─ 判断改动级别
  │   ├─ 🟢 微小 → 跳过（不存）
  │   ├─ 🟡 小 → 存快照 + KB
  │   ├─ 🔴 大 → 存快照 + KB + 更新文档
  │   └─ 🏁 里程碑 → 全套 + 全局归档
  │
  └─ 完成！
```

---

## 💡 关键原则

1. **不打扰用户**：做完事**自动**收尾
2. **不过度操作**：小改动不要触发全套流程
3. **保文档准确**：大改动**必**更新文档
4. **可追溯**：每个 KB、每个快照都有意义
5. **可回滚**：归档有 Git tag

## 🚨 失败路径（v1.6.0 新增）

> **作用**：定义系统异常时的降级策略，让 AI 在出错时知道**回退**而非**硬扛**。

### 失败决策树

```
测试失败？
  ├─ 是 → 不存快照，先修 bug（回到开发循环）
  └─ 否 ↓

快照保存失败？
  ├─ 是 → 检查 00_ROOT_快速加载会话.md 是否可写
  │       ├─ 可写 → 重新跑 save.js
  │       └─ 不可写 → 跳过索引更新，但快照文件本身必须存到 .claude/snapshots/
  └─ 否 ↓

LLM 评分失败？
  ├─ 是 → 自动降级到 heuristic 评分（永不抛错给用户）
  └─ 否 ↓

dispatcher 决策错误？
  ├─ 是 → 测试套件会暴露（test-dispatcher.js 跑挂）
  │       → 改坏 RULES 不会被静默放过
  └─ 否 ↓

worktree 合并冲突？
  ├─ 是 → 不强制 merge，提示用户手动解决
  └─ 否 → 正常完成
```

### 降级原则

| 场景 | 降级策略 | 实现位置 |
|:-----|:---------|:---------|
| LLM 不可用 | 降级到启发式评分 | `llm-adapter.js#scoreWithFallback` |
| ROOT 索引损坏 | 跳过索引更新，保留快照 | `save.js` try/catch |
| dispatcher 损坏 | 测试套件暴露，不静默通过 | `test-failure-paths.js` |
| 测试本身失败 | 不存快照，先修 | `自我约束规范.md` 决策树 |
