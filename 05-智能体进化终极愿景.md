```

  ↓  性能 / 错误追踪 / 用户反馈
```

## 🎯 关键判断 / 设计原则

### 1. 不重新发明轮子

> **所有 UI 框架 / DevOps 工具 / 数据库 ORM 都用市面主流**，不自己造。Claude 的价值在于**组合 + 自动化 + 质量评估**，不在于发明。

### 2. 编码强项别再优化

> M12/M14/M15 已经让编码能力到顶。把精力放在 UI / DevOps / 数据库 / 设计感这 4 个新维度。

### 3. 设计感是 AI 弱项，必须外挂 + 过滤

> **接入 v0.dev 是 ROI 最高的捷径**。同时建立 LLM-judge 设计评估系统，自动化保证质量。
>
> **额外**：用户反馈抖音/小红书刷到的"宝藏 skill / 提示词合集"90% 是营销号低质内容。**M36C skill-quality-filter** 用 LLM-judge 评分 ≥ 7 才入队，防污染。

### 3.5 宝藏 skill / 提示词的自动化路径（用户新需求 · 2026-06-28）

> **用户洞察**：抖音/小红书/X 上常看到"AI 编程宝藏 skill"项目（如 awesome-claude-skills / prompt-Engineering-Guide / awesome-ai-agents）。这类项目**已被 github-scanner 关键词 20 个覆盖**，LLM-judge 自动评分。但需要 3 个增强：

1. **M36C skill-quality-filter** —— 评分 ≥ 7 才入队，防营销号低质内容
2. **M37 数据源扩展** —— 加 douyin/xiaohongshu 关键词监听，用户刷到好东西直接对话告诉我
3. **M40 复用 M36C** —— ui-design-judge 同时过滤低质提示词，避免变成"提示词调包侠"

**自动化路径**：
```
用户对话提到宝藏 skill URL
  ↓ memos.remember("发现宝藏: ...")
  ↓ memos 去重（同一 URL 只存一次）
  ↓ 下次 /evolve 扫描自动命中
  ↓ LLM-judge 评分（M36C 过滤）
  ↓ 高质量入 candidates.json
  ↓ 用户审批 → 落库到 skill 库
```

### 4. 万能 ≠ 全平台

> **v4.0 万能智能体 MVP 边界 = Web 全栈**（Next.js + shadcn + Tailwind + 数据库 + DevOps）。**桌面 / 移动延后到 v5.0**——否则摊子铺太大。

### 5. 渐进交付，每个阶段可独立发布

> 每个里程碑都是"可用增量"，不是"半成品等下一阶段"。v4.0 发版前可以持续发 v3.x 修补。
