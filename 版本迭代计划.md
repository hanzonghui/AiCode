## 二、核心系统设计

> 详细设计见：[自我进化循环系统设计.md](./自我进化循环系统设计.md)

### 2.1 自我进化循环

```
GitHub Trending + Search API
  ↓
trending.json（每日快照，保留7天）
  ↓
评估引擎（规则 + LLM 打分）
  ↓
candidates.json（评分 + 理由 + 复杂度）
  ↓
实现引擎（创建分支 → 写代码 → 测试 → 审查）
  ↓
QA 验证 + 合并 + 左脑记忆 + evolved-features.json
  ↓
持续感知层（每日/每周/月度对比，防退化）
  ↓
发现更好方案 → 升级建议 → 回到实现流程
```

### 2.2 评估维度

| 维度 | 权重 | 标准 |
|:-----|:-----|:-----|
| 实用性 | 30% | 对工作流有什么帮助 |
| 可行性 | 25% | 现有架构能否实现 |
| 独立性 | 20% | 是否依赖外部服务 |
| 风险度 | 15% | 引入后是否破坏现有系统 |
| 新鲜度 | 10% | 是否是我们还没有的能力 |

### 2.3 决策规则

- `composite >= 7.0` → 🟢 自动采纳（P0）
- `composite 5.0-6.9` → 🟡 建议采纳（需确认）
- `composite < 5.0` → 🔴 跳过

### 2.4 安全约束

- ⛔ 不引入 npm 依赖（除非绝对必要）
- ⛔ 不修改 dispatcher.js 核心
- ⛔ 不删除现有文件
- ✅ 必须通过 `npm test`
- ✅ 新功能必须带测试
- ✅ 新功能必须更新文档

## 六、路线图

```
现在 ── v1.8 已完成并推送 GitHub
  ↓
观察期 ── 用 2-3 天，看哪些功能实际被用到
```
